## 1. 目的・スコープ

Automotive SPICE SWE.1〜SWE.6 の各工程ドキュメントを **作成・編集・閲覧・追跡** するための Windows 向けデスクトップウェブアプリケーション。  
本書はそのソフトウェア要件を定義する。

**対象外**: ドキュメントの印刷・PDF 出力、外部ツール連携

---

## 2. 機能要件

| SWR-ID   | タイトル                          | 説明                                                                 | 優先度 | 受入基準                                   |
|----------|-----------------------------------|----------------------------------------------------------------------|--------|--------------------------------------------|
| SWR-001  | ポータル表示                      | SWE.1〜6 各工程の一覧と各工程に属するドキュメントのステータスを表示する | Must   | 全6工程が画面に表示され、ステータスバッジが見える |
| SWR-002  | ドキュメント作成                  | 各工程のドキュメント型に対応したテンプレートを選んで新規ドキュメントを作成できる | Must | 選択した型のフォームが開き、保存後にファイルが生成される |
| SWR-003  | ドキュメント編集                  | 既存ドキュメントの全フィールドを編集できる                           | Must   | フォームの全フィールドを更新して保存できる     |
| SWR-004  | ドキュメント閲覧                  | ドキュメントを読み取り専用の整形済みビューで表示できる               | Must   | 全フィールドが見やすく表示される               |
| SWR-005  | 項目の追加・削除・並び替え        | 要件リスト等の配列フィールドで行の追加・削除・ドラッグ並び替えができる | Must | 追加ボタンで行が増え、ドラッグで順序変更できる  |
| SWR-006  | アンドゥ・リドゥ                  | 編集中の操作を少なくとも 50 ステップ取り消し・やり直しできる         | Must   | Ctrl+Z/Ctrl+Y で操作が可逆である              |
| SWR-007  | ワークスペース記憶                | ドキュメントフォルダを初回設定後に記憶し、次回起動時も再指定不要にする | Must  | 再起動後に前回のドキュメントフォルダが開く     |
| SWR-008  | タスクバー/ブックマーク起動       | Windows タスクバーへのピンまたはブラウザブックマークから起動できる   | Must   | bat ファイルをタスクバーにピンでき、クリックでブラウザが開く |
| SWR-009  | 確認ダイアログ排除                | 保存・削除等の通常操作で確認ダイアログを表示しない                   | Must   | 保存・削除に確認ダイアログが出ない             |
| SWR-010  | トレーサビリティマトリクス表示    | 上位要素から下位要素への対応関係をマトリクス表で表示する             | Must   | SWR → TC の対応が一覧で確認できる            |
| SWR-011  | 影響範囲表示                      | 上位ドキュメントが変更された場合に影響を受ける下位ドキュメントをハイライトする | Must | 変更フラグ付きドキュメントの関連先が橙色で表示される |
| SWR-012  | レスポンシブレイアウト            | ブラウザウィンドウサイズに合わせて各パネルが自動的にリサイズされる   | Must   | ウィンドウを変更するとレイアウトが追従する     |
| SWR-013  | 工程完了状況の把握                | 各工程で必須ドキュメントが揃っているかを確認できる                   | Should | 必須ドキュメントの充足率がパーセントで表示される |
| SWR-014  | ドキュメント検索                  | ドキュメントタイトル・ID でインクリメンタル検索できる                | Should | 文字入力で候補が絞り込まれる                   |

## 3. 非機能要件

| SWR-ID   | 種別     | 要件                                      | 目標値                     |
|----------|----------|-------------------------------------------|---------------------------|
| SWR-N01  | 起動時間 | サーバー起動完了からポータル表示まで       | 5 秒以内                   |
| SWR-N02  | 対応 OS  | Windows 10 / 11 で動作する                | 両バージョンで検証済み      |
| SWR-N03  | 対応ブラウザ | Chrome 120+ / Edge 120+ で動作する     | 両ブラウザで検証済み        |
| SWR-N04  | 依存関係 | Node.js 18+ のみを前提とする              | npm install 1 コマンドで完了 |
| SWR-N05  | 保守性   | コード行数 200 行を超えるファイルは分割する | 全ファイルが 200 行以内     |

---

## 4. 制約条件

- 実装言語: Node.js (バックエンド) + Vanilla HTML/CSS/JS (フロントエンド)
- フレームワーク: Express.js (サーバー)、外部 UI フレームワーク不使用
- ドキュメント保存形式: JSON ファイル (1 ドキュメント = 1 ファイル)
- サードパーティ CDN: marked.js (Markdown レンダリング) のみ許容

---

## 5. トレーサビリティ

本ツールはシステム要件なし (独立プロダクト) のため SYS 要件との対応なし。  
→ SWE.2 アーキテクチャ設計へ割り当て: SWR-001〜014 → ARC-001〜006 参照。
