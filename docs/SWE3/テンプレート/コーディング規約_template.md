## 2. ファイル構成・命名規則

### 2.1 ディレクトリ構成

```
src/
  <モジュール名>/
    <モジュール名>.c
    <モジュール名>.h
include/
  <公開ヘッダ>.h
test/
  <テスト対象名>_test.c
```

### 2.2 ファイル命名

| 種別     | 命名規則                   | 例              |
|----------|---------------------------|-----------------|
| ソース   | スネークケース、小文字      | motor_control.c |
| ヘッダ   | ソースと同名               | motor_control.h |
| テスト   | 対象名 + `_test`           | motor_control_test.c |

---

## 3. コーディングスタイル

| 項目         | ルール                              |
|--------------|-------------------------------------|
| インデント   | スペース4文字（タブ禁止）            |
| 行長         | 最大120文字                         |
| 中括弧       | K&R スタイル（開き括弧は同行）      |
| 空行         | 関数間に1行、論理ブロック間に1行    |

---

## 4. 命名規則

| 種別             | 規則                         | 例                  |
|------------------|------------------------------|---------------------|
| 変数（ローカル） | スネークケース、小文字        | `sensor_value`      |
| 変数（グローバル）| `g_` プレフィックス          | `g_system_state`    |
| 定数・マクロ     | UPPER_SNAKE_CASE             | `MAX_RETRY_COUNT`   |
| 関数             | スネークケース、動詞始まり    | `get_sensor_value()`|
| 型定義 (typedef) | パスカルケース + `_t`        | `SensorData_t`      |
| 列挙値           | `<型名>_` プレフィックス    | `STATE_IDLE`        |

---

## 5. コメント規則

### 5.1 ファイルヘッダ

```c
/**
 * @file    <ファイル名>
 * @brief   <1行の説明>
 * @version <バージョン>
 * @date    YYYY-MM-DD
 */
```

### 5.2 関数コメント

```c
/**
 * @brief   <関数の目的>
 * @param   <引数名>  <説明>
 * @return  <戻り値の説明>
 * @note    <特記事項（オプション）>
 */
```

### 5.3 インラインコメント

- コードの「なぜ」を説明する（「何をするか」はコードが語る）
- 行末コメントは最大40文字

## 6. 禁止事項

| 禁止項目             | 禁止理由                       | 代替手段                     |
|---------------------|-------------------------------|------------------------------|
| `goto` 文           | 制御フローが不明瞭             | ループ・早期リターン          |
| 動的メモリ割り当て   | ヒープ断片化・未解放リスク     | 静的割り当て・メモリプール     |
| グローバル変数の多用 | 副作用・テスト困難             | 引数・戻り値で受け渡し        |
| マジックナンバー     | 意味不明・変更困難             | 名前付き定数・マクロ          |
| `printf` (組込み)   | 実行時間不定・スタック消費     | 専用ログ API                  |
| 再帰呼び出し        | スタックオーバーフローリスク   | イテレーティブな実装           |

---

## 7. セキュリティ考慮事項

| 項目           | 対策                                            |
|----------------|-------------------------------------------------|
| バッファオーバーフロー | 配列アクセス前に境界チェックを必ず実施    |
| 整数オーバーフロー | 演算前に範囲チェック、符号無し型を優先使用 |
| 初期化漏れ     | 変数宣言と同時に初期化する                      |
| 入力検証       | 外部入力はすべて範囲・型チェックを実施          |
