# 作業方針 — Automotive SPICE SWE ドキュメントルール策定プロジェクト

## 1. 目的

Automotive SPICE (ASPICE) の SWE.1〜SWE.6 各工程において、  
プロジェクト内で統一して使用するドキュメントの **作成ルール・テンプレート** を策定する。

これにより以下を実現する。

- 各工程で「何を・どのように書くか」を標準化し、品質のばらつきを排除する
- アセスメント時に根拠となる成果物を確実に揃える
- 工程間のトレーサビリティを確保し、変更影響分析を容易にする

## 4. ルール策定の方針

各工程の `ルール.md` には以下の7項目を必ず記述する。

| 項番 | 項目                   | 内容                                                   |
|------|------------------------|--------------------------------------------------------|
| 1    | プロセス概要           | ASPICE 定義に基づく目的・位置づけ・入出力              |
| 2    | ベースプラクティス     | 該当工程の BP 一覧と各 BP が求める成果物・活動         |
| 3    | 必須ドキュメント一覧   | 作成が必要なドキュメント名・略称・目的                 |
| 4    | ドキュメント記述ルール | 必須記載項目・形式・品質基準（各ドキュメントごと）     |
| 5    | トレーサビリティ要件   | 上位/下位工程との追跡可能性の確保方法                  |
| 6    | レビュー・承認基準     | レビュー観点・チェックリスト・承認条件                 |
| 7    | 成果物管理ルール       | ファイル命名規則・バージョン管理・変更管理手順         |

---

## 5. 共通原則

1. **ASPICE 準拠**: Automotive SPICE Process Reference Model (PRM) ver.3.1 に準拠する
2. **トレーサビリティ**: 要件 → 設計 → 実装 → テスト の双方向追跡を必須とする
3. **一意識別**: すべての要件・設計要素・テストケースには一意の ID を付与する
4. **変更管理**: ドキュメントの変更は変更履歴欄に記録し、承認を得てから反映する
5. **レビュー必須**: すべての成果物はリリース前に少なくとも1名の独立したレビューを受ける
6. **ツール非依存**: テンプレートは Markdown 形式とし、特定ツールへの依存を避ける
